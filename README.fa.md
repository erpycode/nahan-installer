## 🎯 نهان چیست؟

**نهان** یک گیت‌وی پروکسی بدون سرور است که روی **Cloudflare Workers** اجرا می‌شود و از پروتکل‌های **VLESS** و **Trojan** پشتیبانی می‌کند. تمام تنظیمات در پایگاه داده **D1** ذخیره می‌شود.

این نصب‌کننده کل مجموعه را به صورت خودکار برای شما مستقر می‌کند — بدون نیاز به ترمینال، بدون پیکربندی دستی، فقط چند کلیک.

---

## ✨ امکانات

| امکان | توضیح |
|-------|-------|
| 🚀 **استقرار با یک کلیک** | نهان را فقط با یک API Token مستقر کنید — بدون نیاز به CLI |
| 🗄️ **پایگاه داده خودکار** | پایگاه داده D1 را به صورت خودکار می‌سازد و پیکربندی می‌کند |
| 🔗 **استقرار خودکار Worker** | کد را آپلود، مستقر و آدرس workers.dev را فعال می‌کند |
| 🌐 **رابط دوزبانه** | پشتیبانی کامل از فارسی و انگلیسی |
| 🔐 **پروکسی امن** | درخواست‌های API از طریق Worker محدود شده relay می‌شوند |
| 📊 **آماده آنالیتیکس** | توکن شامل دسترسی آنالیتیکس برای مانیتورینگ است |

## 📋 چه چیزی ساخته می‌شود؟

وقتی نصب‌کننده را اجرا می‌کنید، منابع زیر در اکانت Cloudflare شما ایجاد می‌شوند:

| منبع | هدف |
|------|-----|
| 🗄️ **پایگاه داده D1** | ذخیره تنظیمات کاربران، قوانین و پیکربندی‌ها |

| 🔗 **اتصال D1** | اتصال پایگاه داده به Worker |
| 🌐 **آدرس workers.dev** | آدرس دسترسی عمومی شما |

## 🚀 شروع سریع

### 🔗 [باز کردن نصب‌کننده](https://erpycode.github.io/nahan-installer/)

نیازی به نصب نیست — فقط لینک را باز کنید و مراحل را دنبال کنید:

1. 🔑 روی **«ساخت API Token»** در نصب‌کننده کلیک کنید — صفحه Cloudflare با تمام دسترسی‌های لازم باز می‌شود
2. 📝 توکن را کپی کنید و در نصب‌کننده وارد کنید
3. ⚙️ نام Worker و پایگاه داده را پیکربندی کنید (شناسه اکانت خودکار دریافت می‌شود!)
4. 🚀 روی استقرار کلیک کنید
5. 🎉 تمام! پنل شما آماده است

> 💡 همچنین می‌توانید آن را به صورت محلی اجرا کنید: مخزن را کلون کنید و `index.html` را در مرورگر باز کنید.

## 🔑 چطوری API Token بسازیم (راهنمای قدم به قدم)

اگر با Cloudflare آشنا نیستید، این مراحل را دنبال کنید:

### مرحله ۱: صفحه ساخت توکن

روی **«ساخت API Token در Cloudflare»** در نصب‌کننده کلیک کنید. صفحه‌ای باز می‌شود که تمام دسترسی‌های لازم از قبل انتخاب شده‌اند.

### مرحله ۲: ساخت توکن

در صفحه Cloudflare:

1. 📜 به پایین صفحه اسکرول کنید
2. 🔵 روی دکمه **«Create Token»** کلیک کنید
3. ✅ دسترسی‌ها از قبل تنظیم شده‌اند — فقط روی **«Continue to summary»** کلیک کنید
4. 🔵 دوباره روی **«Create Token»** کلیک کنید
5. 📋 **توکن را کپی کنید** (رشته‌ای طولانی از حروف و اعداد)
6. 📝 آن را در نصب‌کننده وارد کنید

### مرحله ۳: تمام!

نصب‌کننده به صورت خودکار اکانت Cloudflare شما را شناسایی می‌کند — نیازی به پیدا کردن دستی Account ID نیست.

> ⚠️ **مهم:** توکن را فوراً کپی کنید! Cloudflare آن را فقط یک بار نمایش می‌دهد. اگر آن را از دست بدهید، باید یکی جدید بسازید.

## 🔒 درباره پروکسی

نصب‌کننده باید از مرورگر به Cloudflare API دسترسی داشته باشد. به دلیل محدودیت‌های **CORS**، دسترسی مستقیم مرورگر به API اغلب مسدود است.

راه حل، یک **Worker پروکسی** (`installer-proxy.js`) است که روی یک اکانت Cloudflare مستقر می‌شود و:
- ✅ فقط مسیرهای مشخص API را قبول می‌کند (D1، Workers، Account)
- ✅ سایر درخواست‌ها را بلاک می‌کند
- ✅ توکن API شما را بدون ذخیره کردن عبور می‌دهد
- ✅ روی لبه Cloudflare اجرا می‌شود (سریع و قابل اعتماد)

**پروکسی پیش‌فرض:** نصب‌کننده از پروکسی میزبانی شده در `nahan-installer.edge1-erpy.workers.dev` استفاده می‌کند.

**میزبانی شخصی:** اگر پروکسی خودتان را ترجیح می‌دهید، `installer-proxy.js` را در اکانت Cloudflare خود مستقر کنید و آدرس URL را در نصب‌کننده به‌روز کنید.

## ⚙️ گزینه‌های پیکربندی

هنگام نصب، می‌توانید پیکربندی کنید:

| گزینه | پیش‌فرض | توضیح |
|-------|---------|-------|
| نام Worker | `nahan-core` | نام Worker مستقر شده |
| پایگاه داده D1 | `iot_db` | نام پایگاه داده D1 |
| رمز پنل | `admin` | رمز ورود پیش‌فرض (بعد از اولین ورود تغییر دهید!) |
| مسیر API | `sync` | مسیر URL داشبورد |
| توکن ربات تلگرام | — | اختیاری: برای اعلان‌ها |
| شناسه ادمین تلگرام | — | اختیاری: شناسه عددی کاربر ادمین |

## 🛠️ میزبانی شخصی پروکسی

اگر می‌خواهید Worker پروکسی خودتان را اجرا کنید:

1. برو به [داشبورد Cloudflare → Workers](https://dash.cloudflare.com/?to=/:account/workers-and-pages/create)
2. Worker جدید بسازید (مثلاً `my-nahan-proxy`)
3. کد پیش‌فرض را با محتوای `installer-proxy.js` جایگزین کنید
4. روی **Save and Deploy** کلیک کنید
5. آدرس Worker را کپی کنید (مثلاً `https://my-nahan-proxy.your-subdomain.workers.dev`)
6. نصب‌کننده را باز کنید و آدرس پروکسی را وارد کنید
